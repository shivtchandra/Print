# Admin Android APK (Capacitor)

Internal **`.apk`** for staff: opens your **hosted** Next.js admin (`/admin`) in a WebView. **No Play Store** — share the APK file directly (Drive, WhatsApp, etc.).

## Prerequisites

- **Node.js** (same as this repo)
- **Android Studio** with Android SDK + a device or emulator  
  https://developer.android.com/studio
- **JDK 17** (or newer). The project is set to compile with **Java 17** so `./gradlew` works with Android Studio’s bundled JDK. If you still see `invalid source release: 21`, in Android Studio go to **Settings → Build, Execution, Deployment → Build Tools → Gradle** and set **Gradle JDK** to **Embedded JDK** or **JDK 17+**.

## One-time setup

```bash
npm install
```

## Point the APK at your live admin

Set the **exact URL** users should see after opening the app (usually login):

```bash
export CAPACITOR_SERVER_URL="https://YOUR_DOMAIN.com/admin/login"
npm run cap:sync
```

- Use **`https://`** in production.
- For testing against **`next dev`** on your Wi‑Fi, use your PC’s LAN IP and **`http://`** (cleartext is enabled automatically), e.g. `http://192.168.1.10:3000/admin/login`.

`cap sync` copies `capacitor.config.ts` into the native project. **Change the URL → run `cap sync` again** before rebuilding the APK.

## Open Android Studio

```bash
npm run cap:open
```

Or open the **`android/`** folder manually in Android Studio.

## Build the APK

1. **Build → Build Bundle(s) / APK(s) → Build APK(s)**  
2. When it finishes, click **locate** in the notification, or find:  
   **`android/app/build/outputs/apk/debug/app-debug.apk`**  
   (Do **not** install the file under `build/intermediates/` — use **`outputs/apk/debug/`** only.)

### Command-line build

```bash
cd android
./gradlew :app:assembleDebug
```

The installable APK is still **`app/build/outputs/apk/debug/app-debug.apk`**.

**Debug APK** is enough for internal use. For a **release** APK (slightly cleaner install prompts), use **Build → Generate Signed App Bundle / APK** and create a keystore once.

## Install on a phone

1. Copy **`app-debug.apk`** to the device.
2. Open the file → allow **Install unknown apps** for Files / Drive / Chrome (Android will prompt).
3. Install → open **Foto Palace Admin** → you should land on your admin URL.

## For staff using the app

- **Keep Wi‑Fi or mobile data on.** The app is a window to your live website; saving products, config, or testimonials only works when the phone **can reach the internet** (and your Vercel URL).
- If something fails to save, check signal and try again after opening the admin in Chrome on the same device.

## Repo layout

| Path | Role |
|------|------|
| `capacitor.config.ts` | App id, name, optional `server.url` from `CAPACITOR_SERVER_URL` |
| `capacitor-www/` | Fallback static page if no `server.url` is set at sync time |
| `android/` | Native Android project (generated by Capacitor) |

## Security reminders

- Prefer **Firebase admin** logins in production; avoid shipping static admin secrets in any client.
- Serve admin only over **HTTPS** for real deployments.
- Treat the APK like a **bookmark + icon**: anyone with the file can try to open your URL — keep admin auth strong.

## Troubleshooting

| Issue | What to try |
|--------|-------------|
| Blank / SSL error | Confirm URL works in Chrome on the same phone. |
| `http://` LAN dev fails | Same Wi‑Fi as PC; firewall allows port 3000; `CAPACITOR_SERVER_URL` uses the machine IP, not `localhost`. |
| Config didn’t apply | Re-run `CAPACITOR_SERVER_URL=... npm run cap:sync`, then rebuild APK in Android Studio. |
| `invalid source release: 21` | Use **Gradle JDK 17+** in Android Studio settings, or run `./gradlew` after pulling the repo (root `build.gradle` forces Java 17 for all modules). |
| Package appears **invalid** | Install **`outputs/apk/debug/app-debug.apk`**, not `intermediates/.../app-debug.apk`. Run `./gradlew :app:assembleDebug` if `outputs` is missing. |
